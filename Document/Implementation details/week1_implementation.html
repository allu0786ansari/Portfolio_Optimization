Week 1 implementation guide: project setup, folder structure, and all source files for the portfolio optimization engine

Deliverable

1
Create project folder

Open PowerShell or Windows Terminal and run these commands one by one.

mkdir portfolio-optimizer cd portfolio-optimizer mkdir data\raw data\processed data\ingestion models backtesting serving dashboard mlops\airflow\dags mlops\monitoring tests .github\workflows

Then initialise Git:

git init git checkout -b main
2
Create virtual environment
python -m venv venv venv\Scripts\activate
Every time you open a new terminal you must run venv\Scripts\activate first. You will see (venv) in your prompt when it is active.
3
Install all Week 1 dependencies
pip install yfinance pandas numpy scikit-learn requests python-dotenv loguru pytest pytest-cov dvc pyarrow fastparquet ta-lib-binary
If ta-lib-binary fails on Windows, run this instead: pip install ta — the code uses the ta library which is pip-friendly on Windows.
4
Create all files

Copy every file from the "All source files" tab into your project. The file path is shown at the top of each code block. Create the files in that exact location relative to your portfolio-optimizer/ root.

5
Initialise DVC (data version control)
pip install dvc dvc init dvc add data/raw echo "data/raw.dvc" >> .gitignore
6
Run the data pipeline
python -m data.ingestion.fetch_data python -m data.ingestion.feature_engineering python -m data.ingestion.macro_features
After running, you should see Parquet files appearing in data/raw/ and data/processed/. Check the verification tab to confirm everything is correct.
7
Run tests
pytest tests/ -v --cov=data --cov-report=term-missing
Target: all 8 tests pass, coverage above 80% for the data module.
8
First Git commit
git add . git commit -m "feat: Week 1 — data pipeline, feature engineering, tests"
1
Check raw data was downloaded
python -c "import os; files = os.listdir('data/raw'); print(f'{len(files)} files in data/raw:'); [print(' ', f) for f in files]"

You should see one Parquet file per ticker — around 40 files total (20 Nifty50 + 20 S&P500 + macro).

2
Inspect a processed feature file
python -c " import pandas as pd df = pd.read_parquet('data/processed/features_RELIANCE.NS.parquet') print(df.shape) print(df.columns.tolist()) print(df.tail(3)) "

Expected output: a DataFrame with ~1250 rows (5 years of daily data) and columns for price, returns, momentum, volatility, and beta features.

3
Check macro features
python -c " import pandas as pd df = pd.read_parquet('data/processed/macro_features.parquet') print(df.columns.tolist()) print(df.tail(3)) "

Expected columns: vix, us_10y_yield, india_vix, sector ETF returns.

4
Run tests with coverage
pytest tests/test_data_pipeline.py -v --tb=short

All 8 tests should pass. If any fail, the error message will tell you exactly which assertion failed.

5
Verify folder structure
python -c " import os for root, dirs, files in os.walk('.'): dirs[:] = [d for d in dirs if d not in ['venv','__pycache__','.git','.dvc']] level = root.replace('.', '').count(os.sep) indent = ' ' * 2 * level print(f'{indent}{os.path.basename(root)}/') for f in files: print(f'{indent} {f}') "
