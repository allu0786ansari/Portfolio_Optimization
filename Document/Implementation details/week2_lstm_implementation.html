1
Install Week 2 dependencies
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu pip install mlflow optuna statsmodels scikit-learn
This installs the CPU version of PyTorch (~200MB). If you have an NVIDIA GPU you can install the CUDA version instead — but CPU is fine for this project size (40 assets, 5 years of data).
2
Create the folder structure for Week 2
mkdir models\forecasting

Then create an empty __init__.py inside it:

type nul > models\__init__.py type nul > models\forecasting\__init__.py
3
Create all 7 source files

Go to the All source files tab and copy each file into your project at the path shown in the header. Create them in this order:

models/forecasting/dataset.py → models/forecasting/lstm_model.py → models/forecasting/arima_baseline.py → models/forecasting/train_forecaster.py → models/forecasting/evaluate.py → tests/test_forecasting.py → models/forecasting/tune.py

4
Start the MLflow tracking server

Open a second PowerShell window (keep your main one open), activate the venv, and run:

venv\Scripts\activate mlflow server --host 127.0.0.1 --port 5000

Then open http://localhost:5000 in your browser — you should see the MLflow UI. Leave this window running.

Every training run will automatically appear here with all metrics and model artifacts.
5
Train the ARIMA baseline first

Go back to your main PowerShell window and run:

python -m models.forecasting.arima_baseline
Expected: ARIMA results logged to MLflow for all 40 tickers. Takes ~3-5 minutes.
6
Train the LSTM model
python -m models.forecasting.train_forecaster
This trains one LSTM per ticker using a walk-forward split. Expect ~15-25 minutes on CPU for all 40 tickers. You will see a progress bar per ticker and metrics logged to MLflow in real time.
7
(Optional) Run Optuna hyperparameter tuning

Only run this after the base LSTM training succeeds. It takes longer (~1 hour) but finds better hyperparameters:

python -m models.forecasting.tune
8
Run Week 2 tests
pytest tests/test_forecasting.py -v
Target: all tests pass. Then commit: git add . && git commit -m "feat: Week 2 — LSTM forecaster, ARIMA baseline, MLflow tracking"
1
Check MLflow has runs logged
python -c " import mlflow mlflow.set_tracking_uri('http://localhost:5000') client = mlflow.tracking.MlflowClient() exps = client.search_experiments() for e in exps: runs = client.search_runs(e.experiment_id) print(f'Experiment: {e.name} — {len(runs)} runs') "

Expected: experiments named arima_baseline and lstm_forecaster, each with 40 runs.

2
Compare LSTM vs ARIMA directional accuracy
python -m models.forecasting.evaluate

Expected output: a summary table showing LSTM directional accuracy > ARIMA for most tickers. Target average: LSTM >55%, ARIMA ~50%.

3
Inspect a saved model artifact
python -c " import mlflow, os mlflow.set_tracking_uri('http://localhost:5000') client = mlflow.tracking.MlflowClient() runs = client.search_runs('1', order_by=['metrics.val_directional_acc DESC'], max_results=1) if runs: r = runs[0] print('Best run:') print(f' Ticker: {r.data.params.get(\"ticker\")}') print(f' Val RMSE: {float(r.data.metrics.get(\"val_rmse\",0)):.6f}') print(f' Val DirAcc: {float(r.data.metrics.get(\"val_directional_acc\",0)):.3f}') print(f' Artifact: {r.info.artifact_uri}') "
4
Run all tests
pytest tests/ -v --tb=short

Should show all Week 1 tests still passing plus new Week 2 tests.
