Before touching code, understand these 4 design decisions — interviewers will ask every one of them.

State space
For each of N assets: log_return, momentum (3 windows), volatility (2 windows), beta, RSI, price_vs_ma50, price_vs_ma200 = 10 features. Plus current portfolio weights (N values). Total: N×10 + N = N×11 floats. Flattened to a 1D vector.
Action space
Continuous vector of N floats in [-1, 1] (Box space). We apply softmax to project onto the simplex — all weights sum to 1, all non-negative. The agent learns raw logits; softmax handles the constraint.
Reward function
Sortino ratio of the episode window, minus a transaction cost penalty proportional to portfolio turnover. We use Sortino (not Sharpe) because it only penalises downside deviation — the agent learns to avoid drawdowns, not just reduce total volatility.
Episode structure
252 trading days (1 year). Random start date sampled from training window on each reset() call. This prevents the agent from memorising a specific market path and forces generalisation across market regimes.
Transaction costs
0.1% per unit of portfolio turnover. Turnover = sum of absolute weight changes. A full reallocation costs 0.1% × 2 = 0.2%. This discourages excessive trading and forces the agent to find stable allocations.
Why not use price directly?
Prices are non-stationary — they trend upward over time. Log returns are stationary and scale-invariant across assets. Using returns makes the state representation consistent across Nifty and S&P500 stocks with very different price levels.
