Four design decisions interviewers will ask about specifically for RL training.

PPO vs SAC — why both?
PPO is on-policy: stable, easy to tune, good first baseline. SAC is off-policy: uses a replay buffer, more sample-efficient, better for continuous actions. We train both and let MLflow's champion-challenger pick the winner objectively.
Timesteps budget
100,000 timesteps for PPO, 150,000 for SAC. Each timestep = one trading day in one episode. 100K steps ≈ 400 episodes of 252 days. This is enough to learn on CPU in ~20 min per agent without overfitting.
Evaluation protocol
After training, run 20 evaluation episodes on the validation window (not seen during training). Average Sharpe across episodes is the champion metric. A new model must beat the current champion by 5% to be promoted.
Markowitz baseline
Mean-variance optimisation using cvxpy with Ledoit-Wolf shrinkage covariance. Maximises Sharpe ratio subject to weights summing to 1, all non-negative, and max single-asset weight of 40%. This is the classical finance benchmark.
Equal-weight baseline
1/N allocation across all assets. Deceptively hard to beat — studies show most active strategies underperform equal-weight over long horizons. If your RL agent can't beat 1/N, that's useful information.
MLflow Model Registry
Every trained model is logged as an artifact. The best model across all algorithms is tagged with alias "champion". The FastAPI server (Week 6) loads by alias — so model promotion is automatic with zero code changes.

5

python -m models.rl_agent.train_agent --algo ppo
You will see episode reward logs every 10,000 steps. Watch the mean_reward trend upward over time — that confirms learning is happening.
6
Train SAC agent (~25 minutes)
python -m models.rl_agent.train_agent --algo sac
7
Run all baselines + compare everything
python -m models.rl_agent.compare_all
This runs Markowitz, equal-weight, and benchmark baselines on the validation window, then prints a full comparison table: RL agents vs all baselines.
