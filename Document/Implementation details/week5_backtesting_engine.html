Walk-forward backtesting is the most interview-critical concept in the entire project. Understand every design decision here.

What is walk-forward backtesting?
Train on window [0, T_train], test on [T_train, T_train+T_test], then slide the entire window forward by one step and repeat. The model never sees future data. Unlike a single train/test split, this tests the model across many market regimes.
Our window parameters
Train: 504 days (2 years). Test: 126 days (6 months). Step: 21 days (1 month forward). With 1241 total days this produces ~11 test windows covering different market conditions: pre-COVID, COVID crash, recovery, rate hike cycle.
Look-ahead bias prevention
Three rules enforced in code: (1) The RL agent is only evaluated on windows it was not trained on. (2) Markowitz covariance is computed from the train window only, then applied forward. (3) No global statistics (mean, std) computed on full dataset.
Risk metrics explained
Sharpe: return / total vol. Sortino: return / downside vol only. Calmar: CAGR / max drawdown. VaR 95%: worst daily loss at 95% confidence. CVaR 95%: expected loss on the worst 5% of days. All annualised where applicable.
Statistical significance test
We run a t-test on monthly excess returns (RL minus equal-weight). p-value < 0.05 means the alpha is statistically significant — not due to random chance. This is what turns a backtest into a credible claim.
HTML report output
The backtest generates a self-contained HTML report with equity curves, drawdown charts, rolling Sharpe, and the metrics table. This is what you screenshot for your README and show during interviews — far more impressive than a terminal printout.
