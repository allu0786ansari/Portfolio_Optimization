Six design decisions interviewers will probe on the serving layer specifically.

Why FastAPI over Flask?
FastAPI is async-native, auto-generates OpenAPI docs at /docs, enforces types via Pydantic at the boundary, and is ~2-3x faster than Flask for I/O-bound workloads. It's the industry standard for ML serving in Python.
Model loading strategy
The champion model is loaded once at startup into memory — not on every request. A background thread polls the MLflow registry every 5 minutes. If a new champion is promoted (by Airflow in Week 9), the API hot-reloads it with zero downtime and zero restart.
API key authentication
Simple bearer token via X-API-Key header. Keys are read from the .env file. Not OAuth2 — that's overkill for a portfolio project. The point is to show you understand that production APIs are not open to the public.
Pydantic validation
All request and response shapes are defined as Pydantic models. FastAPI validates inputs automatically — invalid ticker lists, malformed dates, or missing fields return a 422 with a clear error message before your ML code ever runs.
Structured logging
Every request logs: timestamp, endpoint, latency_ms, status_code, and key input/output values as JSON. In production, these logs feed into CloudWatch or Datadog. Structured logs are searchable; print() statements are not.
/health vs /ready
/health returns 200 immediately — confirms the process is alive. /ready returns 200 only when the model is loaded — used by Docker health checks and load balancers to gate traffic. A container can be alive but not ready if the model is still loading.

Install Week 6 dependencies
pip install fastapi uvicorn[standard] httpx
fastapi is the framework, uvicorn is the ASGI server that runs it, httpx is needed for FastAPI TestClient in integration tests.
2
Add API key to your .env file
echo API_KEY=portfolio-secret-key-change-in-production >> .env

This is the key you'll pass in the X-API-Key header when calling the API.

4
Start the API server

Open a new terminal window (keep MLflow running in its own), activate venv, then:

venv\Scripts\activate uvicorn serving.main:app --host 0.0.0.0 --port 8000 --reload
Open http://localhost:8000/docs — you'll see the auto-generated Swagger UI with all endpoints. This is what you demo in interviews.
