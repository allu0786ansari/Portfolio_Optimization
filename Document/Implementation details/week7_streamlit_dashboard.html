Live public dashboard on Streamlit Cloud

Four pages — each tells a different story about the project to the interviewer.

Page 1 — Performance Overview
The headline page. Shows 6 KPI metric cards (Sharpe, Sortino, CAGR, Max Drawdown, CVaR, alpha significance), the equity curve vs all baselines, drawdown chart, and rolling 63-day Sharpe. Reads from backtest_results.json. This is the first thing an interviewer sees — it needs to communicate "this model works" in under 5 seconds.
Page 2 — Asset Allocation
Interactive allocation tool. User selects tickers from a multiselect, clicks "Get Allocation", and the dashboard calls your live FastAPI /predict endpoint in real time. Shows the returned weights as a donut chart and a ranked bar chart. This is the live demo moment — showing a real API call returning real predictions.
Page 3 — Strategy Comparison
Side-by-side table comparing RL agent vs Markowitz vs Equal-Weight across all 7 metrics. Also shows per-window Sharpe bar chart from the walk-forward backtest — makes it clear the RL agent was evaluated across multiple market regimes, not just one lucky period.
Page 4 — Efficient Frontier
Markowitz efficient frontier scatter plot showing the risk-return tradeoff for random portfolios vs the optimal frontier. Plots your RL agent's average risk/return point on the same chart. Shows financial domain knowledge — not just ML engineering.

Files in order:

dashboard/app.py → dashboard/data_utils.py → dashboard/pages/1_Performance.py → dashboard/pages/2_Allocation.py → dashboard/pages/3_Comparison.py → dashboard/pages/4_Frontier.py

4
Make sure both servers are running

You need two terminals running simultaneously:

Terminal 1: mlflow server --host 127.0.0.1 --port 5000 Terminal 2: uvicorn serving.main:app --host 0.0.0.0 --port 8000 --reload
5
Launch the dashboard
streamlit run dashboard/app.py
Opens automatically at http://localhost:8501. You'll see the sidebar with 4 page links and the Performance page loaded by default.

1
Create requirements.txt for Streamlit Cloud
pip freeze > requirements.txt
After generating, open requirements.txt and remove any line containing torch==2.11.0+cpu — the +cpu suffix breaks pip on cloud. Replace it with just torch==2.11.0
2
Create .streamlit/config.toml
mkdir .streamlit type nul > .streamlit\config.toml

Paste this into .streamlit/config.toml:

[theme] primaryColor = "#185FA5" backgroundColor = "#FFFFFF" secondaryBackgroundColor = "#F5F5F5" textColor = "#1A1A1A" [server] headless = true port = 8501
3
Push to GitHub
git add . git commit -m "chore: add requirements.txt and streamlit config for cloud deploy" git push origin main
4
Deploy on Streamlit Community Cloud

Go to share.streamlit.io → Sign in with GitHub → New app → Select your repo → Set main file path to dashboard/app.py → Deploy.

The cloud deploy shows Performance and Comparison pages (reads from JSON). The live Allocation page needs your FastAPI running — on cloud you can point it to your EC2 URL once deployed in Week 8.
