Docker + GH Actions
Deliverable

1
Install Docker Desktop for Windows

Download from docker.com/products/docker-desktop. After install, open Docker Desktop and wait for the whale icon in the system tray to turn green.

In Docker Desktop → Settings → General: enable "Use WSL 2 based engine". Required for Windows.
2
Verify Docker is working
docker --version docker compose version
Expected: Docker version 27.x.x and Docker Compose version v2.x.x
3
Create folder for GitHub Actions
mkdir .github\workflows
4
Create all 8 files from the source files tab

Click All source files above and copy each file to its path. Order:

serving\Dockerfile dashboard\Dockerfile docker-compose.yml .dockerignore .github\workflows\ci.yml .github\workflows\cd.yml pyproject.toml (merge ruff config) Makefile (replace with updated version)
5
Generate requirements.txt
pip freeze > requirements.txt
Open requirements.txt and change torch==2.11.0+cpu to torch==2.11.0 — the +cpu suffix breaks pip inside Docker.
6
Build and start the full stack
docker compose build docker compose up
First build takes 8-12 minutes — downloads base images and installs all packages. Subsequent builds are fast due to layer caching.
7
Set up GitHub Secrets for CD

Go to your GitHub repo → Settings → Secrets and variables → Actions → New repository secret:

DOCKER_HUB_USERNAME your_dockerhub_username DOCKER_HUB_TOKEN your_dockerhub_access_token API_KEY portfolio-secret-key-change-in-production

Get your DockerHub token at hub.docker.com → Account Settings → Security → New Access Token

8
Push to GitHub and watch CI run
git add . git commit -m "feat: Week 8 — Docker, docker-compose, GitHub Actions CI/CD" git push origin main
Go to your repo → Actions tab. Watch lint → test → build run. Green checkmarks = CI passing.
1
Check all containers are running
docker compose ps NAME STATUS PORTS portfolio-mlflow healthy 0.0.0.0:5000->5000/tcp portfolio-api healthy 0.0.0.0:8000->8000/tcp portfolio-dashboard healthy 0.0.0.0:8501->8501/tcp
2
Test API from outside the container
curl http://localhost:8000/health curl http://localhost:8000/ready
Both should return JSON with status ok/ready. Then open http://localhost:8501 for the dashboard.
3
Check image sizes
docker images | findstr portfolio

API image should be under 1.5GB. If larger, the multi-stage build may not have run correctly.

4
View logs per service
docker compose logs api --tail=30 docker compose logs dashboard --tail=20
5
Stop everything cleanly
docker compose down
